### What was done

**Canonical type reconciliation (per the deferred-work spec):**

### The problem

The six PRs were not a clean stack — three of them (#64, #65, #70) each

ProviderLogo*, UsageBar, ExtraUsageStatusPill, ResetCountdown*), defining the

pick one coherent architecture per axis, keep every unique behaviour, drop

## Session 6: Finish the org API-usage feature (instead of deleting it)

**Status:** Complete — built on `integrate/all-prs`, folded into PR #74
**Decision:** Vincent chose to *finish* the admin/API-usage providers rather
than accept #65's removal — he'll have users on the API model. Key realisation:
the old fetch was **real** (per-model 7-day org token usage, paginated from
both admin endpoints); only the *display* was broken — it stuffed real tokens
into a quota bar with a fake denominator (`max(tokens×1.5, 1M)`). So the fix is
a proper spend card, not a quota card.

### What was built (new, clean-room on the new architecture)

- **`ApiProvider`** (`.anthropic` / `.openai`) — a concept **separate from the
  subscription `ServiceType`**, so API providers never leak into subscription
  card/quota logic (the original design flaw).
- **`ApiUsage` / `ApiModelUsage` / `ApiUsageWindow`** — spend + tokens + per-
  model breakdown over a window; no quota/reset. `ApiUsagePricing`: self-
  contained Anthropic + OpenAI per-million rates → **real** estimated $.
- **Restored + adapted services**: `KeychainManager`, `AuthenticationManager`
  (keyed by `ApiProvider`), `ApiUsageService` (the real org-usage fetch, now
  returning `ApiUsage` priced from the per-model tokens), and a re-added
  `ServiceSupport.fetchDecoded`.
- **`ApiUsageStore`** — `@MainActor` store holding usage per provider + the
  selected window; refetches on window change / refresh; only fetches
  providers with a key entered.
- **`ApiUsageCard` + `ApiUsageSection` + `ApiUsageWindowPicker`** — spend card
  (logo, $ headline, tokens, top-model rows) with a **7d / 30d / Custom** window
  selector (Custom reveals from/to date pickers). Renders in **both** the
  popover (compact) and the dashboard (when a key is entered).
- **Settings**: "API Usage (organization)" section — admin-key `SecureField`s
  for Anthropic/OpenAI with Save/Remove/Help + connected status; saving
  triggers a refresh.
- **Plumbing**: `ProviderLogoKind.openai`, `MeterBarTheme.openaiAccent` /
  `accent(for: ApiProvider)`.
- **Tests**: `ApiUsageTests` — pricing per model (incl. opus-4 reduced rate and
  provider-default fallback), window ranges (incl. custom-order normalisation),
  and DTO decoding for both providers.

Verified: `swift build`, `MeterBarCLI` build, `swiftlint --strict` (0). The app
+ widget `xcodebuild` and `swift test` run on CI.

### Net effect

The admin-API code is **kept and finished**, not deleted-then-re-added: MeterBar
now shows subscription **quota** cards and organization **API-spend** cards side
by side, each with the right model for its data.
