## Session 1: MeterBarShared extraction (deferred work §1)

**Duration:** ~1 hour
**Status:** Complete (verified locally except xcodebuild; CI covers app+widget build)

### What was done

Created `Packages/MeterBarShared` (local SwiftPM package, tools 5.9, macOS 13
min so `MeterBarCLI` — the lowest-targeting consumer — can depend on it) holding
the canonical wire-format types, and deleted the three divergent forks.

**Canonical type reconciliation (per the deferred-work spec):**

- `UsageLimit`: kept `windowSeconds: TimeInterval?` (app), absorbed the widget's
  `clampedUsed`/`clampedTotal` helpers, `used`/`total` stay `Double` (the old
  CLI `Int` decode bug class is now structurally impossible).
- `ServiceType`: `iconName` = SF Symbol (app usage), new separate `assetName` =
  asset-catalog name (widget usage), `sortOrder` moved onto the canonical type.
- `UsageMetrics`: kept `extraUsage` + `resetCreditsAvailable`, so the widget no
  longer silently drops them on decode.
- `UsageStatus`: extracted **without** SwiftUI `Color` (package is
  Foundation-only). The app never used `UsageStatus.color` (views compute their
  own colors); the widget defines a local `extension UsageStatus { var color }`.
- `UsagePace`, `UsageDurationText`, `PaceLabelContext` moved along with
  `UsageLimit` (its `pace()`/countdown helpers depend on them).

**Consumers wired up:**

- App: root `Package.swift` depends on the package; 16 app files got explicit
  `import MeterBarShared` (SWIFT_UPCOMING_FEATURE_MEMBER_IMPORT_VISIBILITY=YES
  means each file needs a direct import); 3 forked model files deleted.
- Widget: ~135 duplicated lines deleted from `UsageWidget.swift`; icon lookups
  switched to `assetName`.
- CLI: dropped its private `ServiceMetrics`/`UsageLimit` fork, now decodes the
  shared `UsageMetrics` directly; provider display names now come from
  `ServiceType.displayName`.
- `MeterBar.xcodeproj` (objectVersion 77): hand-added
  `XCLocalSwiftPackageReference` + two `XCSwiftPackageProductDependency`
  entries + `PBXBuildFile` productRefs in both Frameworks phases, and
  `packageReferences` on the project — MeterBar app and MeterBarWidgetExtension
  targets both link `MeterBarShared`.

**Tests:** added `MeterBarTests/CachedMetricsContractTests.swift` covering the
`cached_usage_metrics.json` contract: round-trip through the shared types, JSON
key stability, legacy-payload tolerance (pre-extraction widget/CLI writers), and
the historical Int-vs-Double regression. Existing test files import the shared
module.
